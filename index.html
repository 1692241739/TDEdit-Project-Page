Abstract

TDEdit is designed for practical interactive editing scenarios where users do not want to choose between semantic control and geometric control. Instead of treating text and drag as two separate pipelines, it models them as two synchronized constraints inside one sampling framework.

Text-guided image editing and drag-guided image editing provide complementary forms of controllability: text prompts specify high-level semantic changes, while drag interactions impose explicit geometric intent. Unifying both within a single inversion-free diffusion framework remains difficult because denoising must preserve dragged structure while still introducing target semantics.

TDEdit retains the inherited three-branch DDCM text-editing path as its semantic backbone and adds two drag-specific components: an intent-aware hybrid 2D/3D drag module that identifies the drag subject, resolves the deformation mode, and encodes edited geometry into the drag denoising anchor z̃0; and a five-branch denoising extension that synchronizes z0 and z̃0 through GSAC and RDM, enabling stable text-only, drag-only, and joint text-drag editing in one framework.

Approach

The core idea is to separate geometry construction from denoising-time semantic editing. This split keeps layout changes explicit and stable, while still letting the model perform high-quality appearance transformation during sampling.

TDEdit separates editing into two stages. Stage 1 transforms the clean source latent z0 into the drag denoising anchor z̃0 under user interaction, so geometry is resolved before diffusion starts. Stage 2 then denoises with dual anchors: z0 preserves source-side appearance cues, while z̃0 preserves the dragged layout.

The inherited source / mutual / reference branches stay anchored at z0, while the drag-aware layout / target branches are anchored at z̃0. Within the drag-aware path, Layout Query Mixing (LQM), Reference KV Mixing (RKM), and Reference Denoise Mixing (RDM) keep dragged geometry and target semantics synchronized during sampling.

In practice, this means the model does not have to repeatedly rediscover the edited structure from noise. The dragged layout is prepared first, then the denoiser spends its capacity on reconciling target appearance, local detail recovery, and source consistency.

Intent-Aware Hybrid Drag Module

The drag module is responsible for interpreting sparse user interaction as an explicit editing instruction. It determines what should move, how it should move, and how the newly exposed background should be repaired before the denoising stage begins.

Intent-aware drag pipeline. SAM2 refines user masks into foreground subjects, grouped handle-target pairs estimate subject-wise deformation, and the drag operator maps z0 to the drag denoising anchor z̃0.

Intent-aware 2D dispatch. The solver selects rigid, non-rigid, or hybrid 2D drag, then parses the in-mode intent and converts it into a unified inverse-warp sampling map.

Depth-aware 3D drag. The pipeline estimates dense relative depth, lifts the selected foreground into 3D, applies one of the 3D drag modes, and projects the result back to 2D with z-buffering and hole filling.

Compared with standard point-drag interfaces, this design makes the user interaction more interpretable. Foreground selection, deformation dispatch, and layout construction are handled explicitly, which reduces ambiguity when the motion is large, non-rigid, or depth-sensitive.
